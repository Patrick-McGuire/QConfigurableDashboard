Detailed Description

Handles coordination and data tunneling across application.

Essentially every QCD object will have a pointer to a common instance of this object. This is how data is transferred between modules and widgets. This also provides a callback system for events. There are a handful of types of data that can be transferred with this object:

  • Json input
    • Json object for passing data from modules to widgets
  • Json output
    • Json object for passing data from widget to modules
  • Json config
    • Application wide config data (unsupported currently)
  • Json theme
    • Theme data (changing at some point)
  • Image input
    • Key-value map of Images for passing from widgets to modules
See also
The way callback events are handled is defined int QCD::CallbackDispatcher
